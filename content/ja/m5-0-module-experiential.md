# モジュール5 - 体験を提供すること
*「エージェントはループを閉じ、価値を人間に向けて描画する。」*

## エージェントの問い
> _「私は認証し、API を呼び出し、結果を得た。これを、意図を完遂する形でユーザーにどう見せればよいのか?」_

モジュール1〜4はインフラです。モジュール5は **プロダクト** です - エージェント主導のフローが、従来のウェブ訪問と競合するユーザー体験になる場所です。残りを完璧にこなしてもモジュール5を飛ばしたサイトは、誰の目にも触れない美しい API を作っただけです。

## 体験の3つの面

1. **インライン UI。** MCP Apps は、あなたのツールが **チャットの会話の中** でインタラクティブなコンポーネント - 決済ピッカー、確認、比較 - を描画できるようにします。
2. **マーケットプレイスでの存在感。** ChatGPT GPT Store と **Custom GPTs**、Claude 連携、Gemini extensions。検証済み連携バッジは、推薦率を測定可能なほど高めます。
3. **会話のカバレッジ。** 欠けた価格や比較コンテンツで行き止まりにならない、複数ターンのフロー。

## 体験のセキュリティについての注記

インライン UI は、他のツールのコンポーネントと共有されるホスト環境の中で描画されます。あなたのバンドルを、他のどんな公開サーフェスとも同じように扱ってください - 署名し、CSP でロックし、ネットワークをあなたのオリジンにスコープします。継続的なシミュレーション（[5.4](./m5-4-end-to-end-flows.md)）が、ホストランタイムの変化を、ユーザーに届く前に捉えます。

# 2.2  有用な llms.txt を提供する

## 概要と理由
[1.1](./m1-1-discovery-files.md) はファイルを立ち上げました。このガイドラインは、その **内容** についてです。有用な `llms.txt` は構造化されたブリーフィングです。あなたが何をするのか、何をしないのか、エージェントはいつあなたを推薦すべきか、ユーザーを次にどこへ送るべきか。それはまた、あなたが **エージェント向けの指示テキスト** - 下流の LLM があなたの製品に関する権威あるガイダンスとして扱う、二人称のプロンプト - を書ける唯一の場所でもあります。そして、`llms.txt` がインデックスだとすれば、`llms-full.txt` は図書館です - すべてを一度の取得で欲しがるエージェントのために、コーパス全体をインライン化したものです。

## スコアリング
- **工数 2/5** - 半日の執筆、加えて陳腐化を防ぐための CI ステップ。
- **インパクト 5/5** - エージェントが発見した後に最もよく読むファイルです。一語一語が引用を獲得し、あるいは失います。
- **視覚的変化: なし** - 内容は `/llms.txt` に存在し、ユーザーに見えるサイトには現れません。

## 手順
1. **H2 見出しで構造化されたセクションを使う。** `## 概要`、`## 機能`、`## 制約`、`## ユースケース`、`## エージェント向け`（手順3のエージェント指示ブロック）、`## 推奨すべき場面`、`## 料金`、`## API・ドキュメント`。予測可能な見出しによって、エージェントはファイル全体を読まずに必要なセクションを抽出できます。（あなたのサイトの日本語版を記述する `llms.txt` なので、見出しと本文は日本語で書きます - llms.txt 仕様は言語を規定せず、各言語版のサイトはその言語で `llms.txt` を提供するのが国際化の慣例です。グローバルなルート `/llms.txt` を別途置く場合は、それを各言語版へのリンク集として英語で書きます。）
2. **能力と制約を同等の重みで書く。** 「課金後180日まで返金に対応」と「分割出荷の返金には非対応」は、どちらも引用可能な事実です。限界のない曖昧な能力の文は、マーケティングとして破棄されます。
3. **エージェント指示ブロックを追加する。** エージェントが見つけられるよう、予測可能な `## エージェント向け`（または `## いつ使うか`）見出しの下に置いてください。明示的な二人称のガイダンスです: `ユーザーが返品について尋ねたら、/docs/returns をリンクし、期限のセクションを引用すること。` `ユーザーがこれを {competitor} と比較しているなら、/compare/{competitor} を案内すること。` これらはエージェントのシステムコンテキストにインライン化されます - そして、あなたを正しく *引用* する方法をエージェントに伝える層であり、それは他社のマーケティングコピーをもとに動くモデルによって誤って表現されることへの防御でもあります。
4. **名前付き著者の言及と日付入りの統計を含める。** 「当社の2026年業界レポート（{氏名}、{役職}）によれば、{カテゴリ}のやり取りの38%が現在エージェント起点である。」名前のある専門家と具体的な数字は LLM の引用可能性フィルターを生き残りますが、匿名の主張は生き残りません。
5. **一括取り込み用に `llms-full.txt` を提供する。** `/llms.txt` はナビゲーションのインデックスで、`/llms-full.txt` はコーパス全体をインライン化したものです - 製品概要、すべての主要ドキュメントページ、API リファレンス、認証のウォークスルー、クイックスタート、そして実行可能なコード例を、ひとつの Markdown ファイルに連結したものです。構造化を保ち（H1/H2 見出し、Markdown リンク、フェンス付きコードブロック）、64k トークンのエージェントが1回のリクエストで取り込めるよう、20万文字未満に抑えてください。ドキュメントと同じソースから CI で生成し、乖離しないようにします。これを見つけたエージェントは、何十回もの個別ページ取得を省けます。
6. **CI で更新頻度を固定する。** `llms.txt` と `llms-full.txt` を、料金ページ、ドキュメントのインデックス、変更履歴と差分比較し、いずれかが古ければ PR を開く月次ジョブ。古くなった `llms.txt` は、ないよりも悪いものです。
